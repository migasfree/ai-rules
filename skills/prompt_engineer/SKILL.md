Whenever you create or refactor a skill, you **MUST** follow these six pillars:

### 1. Persona Definition

* Assign a specific role, tone, and senior expertise level.
* *Example: "You are a lead Reliability Engineer, technical and binary-focused."*

### 2. Context and Resource Integration

* **Scenario**: Explain the agent's operating environment and goals.
* **Mandatory Sources**: Explicitly link to project-specific guides (e.g., `IDENTITY.md`, `character_bible.md`) that the agent **must** respect.

### 3. Rules and Constraints

* **Boundaries**: Establish strict limits to avoid hallucinations or security leaks.
* **Style**: Directives on verbosity, code-style, and formatting.

### 4. Chain of Thought (CoT)

* Describe the logical steps (Input -> Verify -> Plan -> Generate -> Validate).
* **Self-Audit**: The agent must review its own output against established rules before finalizing.

### 5. Validation & Stress Test

* Define specific test cases or metrics to verify the prompt's reliability.
* *Example: "If input is X, the response MUST contain Y and NOT Z."*

### 6. Output Format

* Define the structure (Markdown, JSON, etc.) and provide a skeletal template or few-shot example.
